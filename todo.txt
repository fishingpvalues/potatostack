- i want this recommend best service free of cost etc and document it how: Forwarding a domain to a local IP address in a FRITZ!Box involves connecting a public domain name to your dynamic public IP address (via Dynamic DNS) and then directing incoming traffic to a specific device inside your network (via Port Forwarding). Here is how to set up this connection: 1. Set Up Dynamic DNS (DynDNS) Since most internet connections have a dynamic IP address that changes, you need a domain that updates to your current IP. Use MyFRITZ! (Built-in): Go to Internet > MyFRITZ! Account to set up a free <name>.myfritz.net address.Use Third-Party (DuckDNS, No-IP, etc.): Go to Internet > Permit Access > Dynamic DNS. Choose "User-defined" and enter your custom domain and update URL. 2. Configure Port Forwarding To direct traffic from the domain to a specific device (e.g., a NAS, home server) on your network: Go to Internet > Permit Access > Port Sharing.Click "Add Device for Sharing" and select the device.Click "New Sharing".Select "Port Forwarding".Set the Application (e.g., HTTP server), and define the internal/external ports (e.g., 80/443 for web services).IPv6 note: If using IPv6, specify the Interface ID of the device manually to ensure it doesn't change. 3. Handle Local Access (DNS Rebind Protection) If you try to access yourdomain.com from inside your home network, the FRITZ!Box may block it to prevent DNS rebinding attacks. Go to Home Network > Network > Network Settings.Scroll down to DNS Rebind Protection.Add your domain (e.g., mydomain.com or *.myfritz.net) to the allowed list. 4. Alternative: Local DNS Server (Pi-hole) If you want to use your domain name internally to point to a local IP without going out to the internet and back, you can set up a local DNS server (like Pi-hole) and set it as the DHCP server in the FRITZ!Box settings. Summary of Key Settings DynDNS: Maps domain.com \(\rightarrow \) Dynamic Public IP.Port Forwarding: Maps Public IP:Port \(\rightarrow \) Internal IP:Port.Rebind Protection: Allows internal resolution of external domains
