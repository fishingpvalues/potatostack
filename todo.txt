# PotatoStack Crash Analysis & Hardening - Action Items

## DIAGNOSIS COMPLETE

**Root Cause Found: OOM (Out of Memory) Kills**

The system crashed due to Immich hitting its 768MB memory limit during video transcoding:
- Jan 25 18:30:40 - OOM killed immich (pid=3931334)
- Jan 25 18:37:15 - OOM killed immich (pid=4006011)
- Jan 25 19:35:24 - OOM killed immich during HEVC transcoding

The crash was NOT caused by tmux - it was memory exhaustion from Immich.

## FIXES ALREADY APPLIED

1. [x] Increased Immich memory limit from 768M to 2G in docker-compose.yml
2. [x] Created enterprise-hardening.sh script at scripts/setup/

## MANUAL ACTION ITEMS

### Step 1: Fix Docker (corrupted from crash)
```bash
sudo systemctl start docker.socket docker
```

If still broken:
```bash
sudo systemctl stop docker.socket docker
sudo rm -rf /mnt/storage/docker/overlay2 /mnt/storage/docker/image
sudo mkdir -p /mnt/storage/docker/overlay2 /mnt/storage/docker/image
sudo systemctl start docker.socket docker
```

### Step 2: Start the stack
```bash
cd /home/daniel/potatostack
docker compose up -d
```

### Step 3: Run enterprise hardening script
```bash
sudo bash scripts/setup/enterprise-hardening.sh
```

This will configure:
- Hardware watchdog (auto-reboot on kernel hang)
- Auto-login on console (optional)
- Network resilience monitor
- Memory pressure handler
- Enhanced health checks every 10 min
- Weekly auto-reboot (optional)
- Kernel parameters for stability

### Step 4: Verify setup
```bash
systemctl status potatostack
systemctl status network-watchdog
systemctl status memory-pressure-handler
systemctl list-timers | grep potatostack
```
